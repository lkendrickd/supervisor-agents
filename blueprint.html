Supervisor Multi-Agent Pattern

LLM-driven orchestration with parallel specialist execution. The supervisor delegates to domain experts via tool calls, synthesizes results into a unified response.

ReAct Agent (Orchestrator)

The supervisor is a standard ReAct agent whose only "tools" are the specialist agents, wrapped as @tool functions. It reads the user's request, decides which specialists to invoke, and can call multiple tools in a single response for parallel execution.

It never sees specialist internals — only the @tool docstrings. After specialists return, it synthesizes a unified answer.

Chained Dependencies: Sequential Execution

When one specialist's output is needed as input for another, the supervisor handles it naturally through its ReAct loop. It calls agents sequentially, threading results between them.

Dependent Query
"Count the words in 'Foo Bar Baz', take that count and multiply by 3"

Supervisor
Analyzes the request, recognizes a dependency chain
reasoning: "The user wants a word count first, then that result used in a multiplication. I need to call Wordsmith first, wait for the count, then pass it to Mathematician."

Key Concepts
Agents-as-Tools Each specialist is wrapped as a @tool function with a simple string-in/string-out interface. The supervisor sees only the docstring, never the internals.
LLM-Driven Parallelism When the supervisor emits multiple tool calls in one response, LangGraph runs them concurrently. The model decides when to parallelize.
Conversation Memory Full message history is maintained across turns. The supervisor and specialists can reference earlier exchanges for context.
MCP Tool Server Tools are served via Model Context Protocol over stdio. One server, tools split across specialists at load time via filter_tools().
Context Isolation Specialists never see the supervisor's routing logic. Each gets a clean context with only its system prompt and the user's request.
Hierarchical Scaling A specialist can itself be a supervisor with sub-agents. Nest graphs for complex domains without changing the top-level pattern.
